┌──────────────────────────────────────────────┐
│ Step 1: Django Defines Schedule              │
│----------------------------------------------│
│ settings.py OR django-celery-beat (DB)       │
│                                              │
│ Example:                                     │
│ - task: send_email                           │
│ - schedule: every 10 sec / cron              │
└──────────────────────┬───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│ Step 2: Celery Beat Scheduler Loop           │
│----------------------------------------------│
│ while True:                                  │
│   for each task:                             │
│     check is_due()?                          │
│                                              │
│ (uses last_run_at + schedule)                │
└──────────────────────┬───────────────────────┘
                       │
          YES (task due) ▼
┌──────────────────────────────────────────────┐
│ Step 3: Task Message Created                 │
│----------------------------------------------│
│ Beat → send_task()                           │
│                                              │
│ Message contains:                            │
│ - task name                                  │
│ - args / kwargs                              │
│ - metadata (eta, retries)                    │
└──────────────────────┬───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│ Step 4: Broker Queue (Redis/RabbitMQ)        │
│----------------------------------------------│
│ Queue:                                       │
│ [ task_1 ][ task_2 ][ task_3 ]               │
│                                              │
│ FIFO (mostly)                                │
└──────────────────────┬───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│ Step 5: Worker Main Process                  │
│----------------------------------------------│
│ - Maintains connection to broker             │
│ - Pulls tasks (consumer)                     │
│ - Pushes to internal queue                   │
└──────────────────────┬───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│ Step 6: Internal Worker Queue                │
│----------------------------------------------│
│ [ task_1 ][ task_2 ][ task_3 ]               │
│                                              │
│ (Prefetched tasks stored here)               │
└──────────────────────┬───────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ Child Proc 1 │ │ Child Proc 2 │ │ Child Proc 3 │
│ (Worker)     │ │ (Worker)     │ │ (Worker)     │
└──────┬───────┘ └──────┬───────┘ └──────┬───────┘
       │                │                │
       ▼                ▼                ▼
┌──────────────────────────────────────────────┐
│ Step 7: Task Execution                       │
│----------------------------------------------│
│ Each process executes:                       │
│ def send_email():                            │
│     business logic                           │
│                                              │
│ Result → backend (optional)                  │
└──────────────────────────────────────────────┘
